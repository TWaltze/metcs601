Functional Specifications

This application was originally developed several years ago, in 2010. Since then though it has not been updated or brought inline with modern day browser capbilities. My proposal is to completely rewrite the application using new technology.

This application will be split into a frontend and a backend. The frontend will use:

  • AngularJS (Javascript)
  • Grunt
  • LESS
The backend will use:
  • Laravel (PHP)
  • MySQL

The frontend will do zero server side processing and get all relevant data through AJAX requests to a REST API that the backend provides. The backend will behave as single source of truth, whereas the frontend will behave more or less as a 3rd party merely consuming the provided API. This gives the ability to easily expand this application to other platforms (iOS, Android, etc) without rewriting various database and request logic.

The application contains 3 main pages:

  • Landing page (Registration and Login)
  • Profile (Drop history)
  • Account settings

Landing Page

Here the user can login if they have an account, or they can create a new account with a username, email, and password. The username they use will also act as their personal URL. The user with the username "tyler" will have their page accessible at http://qikdrop.com/tyler. After logging in or creating a new account, they will be directed to their profile page.

Profile Page

This is where most of a user's actions are contained. On this page, a user is presented with a text box, a file upload input, and a linear stream. In this respect, it would look similar to Twitter's profile pages. A user can choose to upload a file and/or post a piece of text, whereas after it would be placed in the stream below. Here, they can also access a search box that will filter their stream based on their search term. Their drop stream will be a linear stack with their newest drops at the top. All profile pages are public by default. A user can also choose to delete individual drops from their account/stream with a single click. Other users can choose to report a user's drop with a single click if they deem it appropriate (Copywrite, illegal, etc).

Account Settings Page

This page will contain various user options:

  • Change your username
  • Change your password
  • Make your profile private

Prototype

The current version of the website can be viewed at QikDrop.com. The general layout of the new version will remain similar, but it will provide improved functionality such as better file upload handling, faster searches, etc. It will also be given a new color scheme and general design language.
